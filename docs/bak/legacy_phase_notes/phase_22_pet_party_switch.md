# Phase22：宠物出战 / 待机 / 休息与换宠

## 本阶段目标

- 给玩家宠物建立战斗内队伍状态，不再只有一个固定的 `ally_pet`。
- 右上角“换宠”按钮打开同一套战斗按钮模板，PC 和手机共用。
- 只有“待机”宠物可以换上场；“出战中”和“休息”宠物不能点。
- 人物选择换宠后，这个回合直接进入行动队列，不再让旧宠物继续选择攻击或技能。

## 当前状态定义

- `battle`：正在场上，占用 `ally.front.3`，当前实现仍复用固定 actorId：`ally_pet`。
- `standby`：待机，可以被换上场。
- `rest`：休息，不能在本场战斗中再被换上场。击飞会把宠物同步到这个状态。

## 参考确认

- 8.0 源码里战斗命令存在 `BATTLE_COM_PETIN` 和 `BATTLE_COM_PETOUT`，说明收回 / 叫出宠物是战斗命令，不是免费 UI 操作。
- 8.0 源码里有待机宠校验，叫出非待机宠会被拒绝。
- 8.0 源码里宠物击飞后会把玩家默认宠物清空。本项目先映射为 `rest`，表示本场战斗不能再放出来。

## 当前可测样本

- 出战中：`我的布伊`
- 待机：`迅捷布伊`
- 待机：`厚皮布伊`
- 休息：`休息布伊`
