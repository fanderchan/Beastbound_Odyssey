# Phase24：战斗异常状态系统

## 目标

Phase24 先把异常状态做成统一的战斗底层机制，而不是只在画面上写一个标记。

本阶段覆盖四类可验证状态：

- `poison`：中毒。不会阻止行动，回合末造成跳伤，并消耗持续回合。
- `sleep`：睡眠。轮到该单位行动时跳过行动，并消耗持续回合；受到物理伤害后解除。
- `confusion`：混乱。攻击类行动会改打同阵营活着单位，并消耗持续回合。
- `stone`：石化。轮到该单位行动时跳过行动，并消耗持续回合；受物理伤害时防御按更高值计算。

这些名字是代码和数据字段，玩家界面只显示中文游戏含义，例如 `毒`、`眠`、`乱`、`石`。

## Actor 状态字段

每个战斗单位都有：

```json
{
  "statuses": {
    "poison": {
      "id": "poison",
      "label": "中毒",
      "turns": 3,
      "potency": 7,
      "sourceId": "ally_player"
    }
  }
}
```

字段含义：

- `id`：状态类型。
- `label`：中文显示名。
- `turns`：剩余持续回合。
- `potency`：状态强度。当前主要给中毒跳伤使用。
- `sourceId`：状态来源，后续服务端账本、技能来源、GM 审计会用到。

## 事件账本

Phase23 的 `battle_event_ledger` 继续作为事实来源。

Phase24 给账本增加：

- `statusId`：本次事件涉及的状态。
- `statusResult`：状态结果，例如 `applied`、`tick`、`skip`、`confused_retarget`。
- `statusChanges`：状态变化明细。
- `targets[].statusesBefore` / `targets[].statusesAfter`：目标事件前后的状态快照。

正常玩家 UI 不显示这些账本字段。工程验证写到 `.run/battle_trace/latest.jsonl`。

## 回合处理顺序

当前本地原型的顺序是：

1. 玩家选择人物行动。
2. 玩家选择自己的宠物行动。
3. 其他友方 NPC 和敌方生成默认行动。
4. 所有行动按速度混合排序。
5. 单个行动执行前检查睡眠/石化。
6. 攻击类行动执行时检查混乱。
7. 伤害命中时处理睡眠解除、石化防御修正。
8. 正常行动队列结束后，生成并播放回合末中毒跳伤。
9. 如果双方仍有活人，进入下一回合命令。

## 当前技能接入

现有 `毒精灵5`、`毒雾精灵5`、`毒粉5`、`毒雾粉5` 现在是：

- 先造成一次即时伤害。
- 如果目标还活着，附加 `poison`。
- `poison` 默认持续 3 回合。
- 每次跳伤为即时伤害的一半，至少 1 点。

这是先用于验证底层链路的临时数值，后续应由技能/道具数据表控制。

## 手工验证

启动：

```sh
godot --path client/godot --scene res://scenes/Main.tscn -- --battle-status-test
```

可观察点：

- `低防乌力` 头顶有 `毒`，回合结束后会跳 `毒 -7`，并减少持续回合。
- `高速乌力` 头顶有 `眠`，轮到它行动时不会攻击，会飘出睡眠状态提示。
- `普通乌力` 头顶有 `石`，轮到它行动时不会攻击，被物理攻击时更抗打。
- `高速猎人` 头顶有 `乱`，当它执行攻击时会改打同阵营目标。
- 睡眠单位被攻击后，`眠` 标记会消失。

## 自动验证

```sh
godot --headless --path client/godot --scene res://scenes/Main.tscn --quit-after 1400 -- --auto-battle-status-check
```

自动验证内容：

- 中毒跳伤扣血并减少回合。
- 睡眠会让行动变成 `status_skip`，不会造成伤害。
- 石化会降低物理伤害。
- 混乱会把攻击目标改成同阵营单位。
- 睡眠在受击后解除。

## 后续暂不做

- 昏睡、麻痹、醉酒、网、深毒等更多原版状态。
- 魔法、道具、宠物技能的完整状态模板编辑器。
- 服务端权威状态结算。
- 原版所有状态动画和特效。

状态命中率、抗性、覆盖、净化和免疫已在 Phase26/27 继续推进。剩余内容应按技能系统、宠物技能数据、服务端战斗权威逐步推进。
