# Phase32：回避 / 反击 / 合击概率 / 幸运一击

## 目标

本阶段把普通物理攻击的“战斗反应”补进事件模型：

- 回避：目标在被普通近战或物理宠物技能攻击时，有概率完全闪开。
- 反击：普通近战攻击结算后，受击方若仍能行动，有概率追加一次反击。
- 合击概率：合击不是无条件触发，而是先做概率门槛，再要求连续同侧同目标普通近战攻击。
- 幸运一击：暴击不是简单翻倍，而是在普通伤害基础上追加一段和目标防御有关的伤害。

这些规则只进入战斗模型、事件账本和自然战斗反馈，不把概率 roll、源码字段名、工程验证文字显示到正常战斗 UI。

## 8.0 服务端源码确认

参考源码来自本地只读研究副本：`.run/reference/StoneAge`，来源为 `fanderchan/StoneAge`。本项目只参考机制，不复制源码或数据。

明确看到的点：

- `gmsv/src/battle/battle_event.c:762-907` 有回避判定。回避和双方敏捷有关，玩家还会加幸运值，最终上限由 `KAWASHI_MAX_RATE` 控制，源码里是 75。
- `gmsv/src/battle/battle_event.c:1619-1646` 的普通攻击序列中，`opt != BATTLE_COM_COMBO` 时才做回避检查。因此合击攻击会跳过普通回避判定。
- `gmsv/src/battle/battle_event.c:1281-1352` 有会心一击概率判定，同样和敏捷、幸运、武器会心修正有关。
- `gmsv/src/battle/battle_event.c:1503-1512` 的会心伤害不是固定 x2，而是在普通伤害基础上追加目标防御、双方等级比相关的额外伤害。
- `gmsv/src/battle/battle.c:4326-4454` 有合击判定。这里不是按左右阵营区分概率，而是按角色类型区分：`CHAR_TYPEENEMY` 基础约 20%，非 enemy 的玩家/宠物基础约 50%；必须是普通攻击、同侧、同目标、可行动、非投掷武器，连续行动者才会合成合击。
- `gmsv/src/include/version.h:395` 里 `_ITEM_ADDCOMBO` 被注释为未完成；相关代码会读取装备参数里的“合击”并追加概率。8.0 这份源码里没有看到敏捷、亲密度或站位直接给合击率加成。
- `gmsv/src/battle/battle_event.c:3603-3755` 有反击判定。反击发生在普通攻击后，不是只在回避后发生；反击伤害会乘 0.75；反击本身也会走攻击序列，所以可以被回避或暴击。
- `gmsv/src/battle/battle.c:7794-7808` 显示原服务端会尝试多段交替反击。Beastbound 第一版先禁止反击继续触发反击，避免早期表现复杂度失控。

## Beastbound 第一版规则

### 回避

- 支持事件类型：`attack`、`skill_attack`、`counter_attack`。
- 不支持事件类型：`combo_attack`、精灵、道具、毒跳伤、异常状态命中。
- 防御中、不能行动、被阻断状态控制时，默认不回避。
- 概率用稳定种子计算，同一次自动测试结果可复现。
- 当前公式是 8.0 风格近似：敏捷差距越大，优势方越容易触发；目标的 `luck`、`dodgeBonus`、`evasionBonus` 可加成；上限 75%。

### 反击

- 只有 `attack` 普通近战会触发反击判定。
- 反击者必须在原攻击结算后仍存活且可行动。
- 反击伤害为自己的普通攻击伤害乘 `0.75`，最低 1。
- `counter_attack` 可以被回避、可以暴击，但不会再次触发新的反击事件。
- 工程测试可用 `counterRateOverride` 强制 0% 或 100%。

### 合击概率

- 普通战斗中，合击先做概率门槛：
  - 玩家/宠物类战斗单位基础 50%。
  - 野怪/敌怪类战斗单位基础 20%。
- 这个区分不是 `ally/enemy` 阵营区分。因此未来 PVP 10V10 里，即使对手站在敌方阵营，只要它是玩家/宠物类型，也走 50% 基础合击率。
- 通过门槛后，才检查连续同侧、同目标、普通攻击、不同攻击者。
- `combo_attack` 不走回避判定。
- 工程测试和动作预览可用 `comboRateOverride` 强制触发。
- 未来装备、被动、活动可以用 `comboBonusRate` 追加概率；如需直接覆盖基础值，可以用 `comboBaseRateOverride`。

### 幸运一击

- 事件字段为 `critical`。
- 支持 `attack`、`skill_attack`、`counter_attack`。
- 当前伤害为：普通伤害 + `目标防御 * 攻击者等级 / 目标等级 * 0.5`，至少比普通伤害多 1 点。
- 未来接入正式等级、武器、幸运、装备会心后，再把公式拆到数据表。

## 新增验证

启动自动检查：

```sh
godot --headless --path client/godot --scene res://scenes/Main.tscn --quit-after 1800 -- --auto-battle-reaction-check
```

检查内容：

- 强制 100% 回避：目标不扣血，显示为回避。
- 强制 100% 反击：普通攻击后追加 `counter_attack`。
- 反击目标强制 100% 回避：反击被闪开，并且不会继续触发反击链。
- 强制暴击：最终伤害高于普通攻击。
- 目标强制 100% 回避时，合击仍然命中。
- 野怪基础合击率为 20%，PVP 敌方玩家/宠物类型基础合击率为 50%，`comboBonusRate` 可以正常追加概率。

## 后续可调点

- 要不要保留 8.0 的“最多 5 次交替反击”原味表现。
- 宠物技能哪些算物理，哪些不能被回避。
- 合击概率是否要在 Beastbound 里原创加入“亲密度、站位、敏捷差、同家族/同种系”修正。8.0 当前确认到的是 enemy/非 enemy 基础区分，以及未完成装备追加率入口。
- 幸运一击是否参与击飞概率加成。
