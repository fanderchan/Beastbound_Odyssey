# Phase125 宠物成长样本基线

本文件记录第一版宠物成长、品质、战力来源样本。目标是让“同种同级宠物为什么有差异”可解释，而不是黑箱随机。

## 生成命令

```sh
godot --headless --path client/godot --scene res://scenes/Main.tscn --quit-after 1800 -- --numeric-experiment-report
```

报告位置：

```text
.run/godot/numeric_experiment_report.json
```

读取字段：

```text
petGrowth.growthSamples
```

## 当前口径

| 口径 | 用途 |
|---|---|
| 战力 | 当前可战斗强度、自动丢弃、宠物排序 |
| 品质 | 个体潜力、极品判断、锁定建议 |
| 成长档 | 物种/形态定位，例如攻击、敏捷、防御、均衡 |
| 成长记录 | 解释最终四维来自基础值、初始加成、成长加成和等级 |

战力公式当前为：

```text
round(maxHp / 4 + attack + defense + quick)
```

品质阈值当前为：

| 标签 | 阈值 |
|---|---:|
| 偏低 | `<= 2400` |
| 普通 | `2401-7599` |
| 偏高 | `>= 7600` |

## 设计判断

- 同一 form、同一等级下，低/中/高个体已经有可见但不过分夸张的战力差。
- 低等级阶段，品质差异存在但不会完全碾压玩法；高等级阶段，成长加成会逐渐拉开长期价值。
- 自动丢弃继续用战力阈值，极品判断和锁定建议应使用品质/成长记录，不要只看战力。
- 转生兽基础值明显高于新手宠，这是任务/转生线的奖励感来源；后续需要通过获得难度和培养成本约束。

## 后续调参入口

- 想改变极品率：优先调 `pet_growth_profiles.json.quality`。
- 想改变同种同级差异：调 `individualVariance.initialBonus` 和 `individualVariance.growthBonus`。
- 想改变某类宠定位：调对应 `growthProfileId` 的 `perLevel`。
- 想改变战力显示口径：调 `powerFormulas`，并同步自动丢弃阈值。
