# Phase128 经济净收入账本基线

本阶段补的是“打完一场到底赚不赚”的账本。之前报告已经能看经验、石币毛掉落和固定战斗胜负，但数值策划真正要看的是净收入：掉落和可卖物品回收，扣掉村医、野外补给储备、装备修理以后，还剩多少循环货币。

这不是最终经济平衡结论，而是后续调怪物、掉落、商店价格、耐久、治疗成本时的固定基线。

## 自测命令

单独跑经济账本：

```sh
godot --headless --path client/godot --scene res://scenes/Main.tscn --quit-after 1800 -- --auto-economy-ledger-check
```

输出报告：

```text
.run/godot/numeric_economy_ledger_report.json
```

总数值报告也会包含同一份 `economyLedger`：

```sh
godot --headless --path client/godot --scene res://scenes/Main.tscn --quit-after 1800 -- --auto-numeric-experiment-report-check
```

## 数据来源

配置表：

```text
client/godot/data/balance/economy_ledger_scenarios.json
```

执行模型：

```text
client/godot/scripts/progression/numeric_economy_ledger_model.gd
```

依赖输入：

- `battle_simulation_scenarios.json` 的固定战斗样本。
- `battle_rewards.json` 的石币和物品期望掉落。
- `item_shops.json` 和 `reward_economy.shop.defaultSellRate` 的出售回收价。
- `player_growth.json` 的村医费用、武器耐久、防具耐久、修理单价。
- `battle_actions.json` 的治疗道具恢复量。

## 账本口径

净收入公式：

```text
净收入 = 石币期望 + 可卖物品期望回收 - 人物村医费 - 野外低血补给储备 - 人物装备修理费
```

当前假设：

| 项目 | 当前值 | 说明 |
|---|---:|---|
| 普通遇敌间隔 | 6.0 秒 | 对齐当前草丛/挂机平均节奏 |
| 每回合耗时 | 2.4 秒 | 作为经济估算，不代表动画最终时长 |
| 结算耗时 | 1.2 秒 | 用于每小时收益估算 |
| 村医费用 | 20 血 / 1 石币 | 读取 `player_growth.json` |
| 武器耐久 | 攻击 100 次掉 1 | 只统计人物普通/物理攻击类事件 |
| 防具耐久 | 被命中 10 次掉 1 | 回避不计，当前只统计人物受击 |
| 修理费用 | 5 耐久 / 1 石币 | 读取 `player_growth.json` |
| 野外补给阈值 | 人物低于 70% 血 | 低于阈值时计一部分战斗治疗道具储备 |
| 补给储备比例 | 35% | 不是每战真实吃药，只是风险准备金 |
| 最便宜战斗治疗 | 0.2857 石币 / 血 | 当前来自肉：8 石币 / 28 血 |

第一版只算主角自身经济，不把陪练伙伴或其他玩家的装备/治疗成本算入主角成本。任务戒指、材料等当前不可卖物品按 0 石币计，但仍保留在 `itemAverages` 里。

## 当前结论

- 第一版经济循环不会因为村医、补给储备和耐久修理立刻变成负收入。
- 新手区净收入最低，但仍能覆盖肉、绳子和基础补给教学。
- 中后期净收入明显偏高，这是未来通胀风险，不是最终平衡目标。后续要靠高阶补给、修理材料、强化、合成、捕捉工具、任务门票等回收，而不是简单把低级怪掉钱砍没。
- 资格战现在按一次性任务样本记录，不能用每小时收益评价，因为它不应该成为刷钱路线。
- 当前材料不可卖，所以它们不会抬高账本收入；以后开放材料出售或拍卖时，经济账本必须重新跑。

## 后续门槛

下一次修改掉落、石币、商店价格、道具治疗量、耐久或修理成本时，至少要满足：

1. `--auto-economy-ledger-check` 通过。
2. 7 个可重复练级段净收入不能为负，除非明确设计成亏损型挑战副本。
3. 新手区净收入必须能覆盖教学补给。
4. 高阶区净收入如果继续高于低阶区几十倍，需要在文档里说明对应金币回收系统。
5. 资格战不能被当作可重复经济来源；如果后续允许复刷，要单独给复刷掉落表。
