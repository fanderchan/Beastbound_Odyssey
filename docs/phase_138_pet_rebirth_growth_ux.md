# Phase138：宠物转生 UX / 成长评价闭环

本阶段把宠物转生后的成长展示从“只看 0 转成长”推进到“按转生阶段评价”。目标是玩家能看懂当前宠物的原生成长、1 转增量、2 转增量，并能通过 MM 转生流程完成一次实际验证。

## 玩家可见闭环

- 宠物详情保留 `个体 / 图鉴 / 成长` 三个主 tab。
- `成长` tab 内部支持阶段：
  - `0转成长`
  - `1转成长`
  - `2转成长`
- 宠物没有达到对应转生阶段时，后续阶段按钮置灰。
- 选中已 1 转宠时，详情文本显示 `1转成长评价` 和 `转生增量`。
- 成长表显示：
  - `初始`
  - `当前`
  - `预测140`
  - `成长/级`
  - `评级`
- 成长雷达图继续显示四项方向：
  - 生命
  - 攻击
  - 防御
  - 敏捷

## 评价规则

0 转成长：

- 以物种成长配置、个体 seed、实际等级成长记录评估。
- 四项显示各自评级。
- 综合评价使用战力成长百分位，而不是简单平均四项。

1 转 / 2 转成长：

- 不再沿用 0 转成长评分。
- 按宠物 `petCultivation.rebirthGrowthBonus` 的转生增量评估。
- 血量增量会按战力口径折算，和攻击、防御、敏捷一起进入综合成长判断。
- 未转生阶段返回 `未开启`，避免玩家误以为转生宠仍按 0 转成长评价。

第一版阈值在 `PetGrowthObservationModel` 内保存：

- 1 转综合增量阈值：`min / p25 / p55 / p85 / p95 / max`
- 2 转综合增量阈值：`min / p25 / p55 / p85 / p95 / max`
- 四项单项增量也有独立阈值。

后续进入正式数值策划时，可以把这些阈值迁到 balance JSON。

## MM 转生流程覆盖

Phase138 依赖后续补齐的 MM 喂石和任务引导，但当前闭环已能验证：

- 获取 1 转 MM。
- 购买并喂石。
- 目标宠和 MM 执行 `转强 / 转生`。
- 转生结果写入 `petCultivation.history`。
- 转生增量进入成长评价。
- 2 转 MM 蛋和一次性 2 转 MM 领取逻辑可验证。
- 第一次 1 转 MM 教学引导会在任务追踪中给出下一步目标。

## 自测入口

```sh
godot --headless --path client/godot --scene res://scenes/Main.tscn --quit-after 2600 -- --auto-pet-growth-observation-check
godot --headless --path client/godot --scene res://scenes/Main.tscn --quit-after 2600 -- --auto-pet-rebirth-mm-check
```

`--auto-pet-growth-observation-check` 重点覆盖：

- GM 获取成长宠。
- 连续升级后能观察成长。
- 手工注入 1 转增量后，成长 tab 切到 `1转成长`。
- `2转成长` 按钮在未 2 转时置灰。
- 成长表、雷达图和详情文本均可见。
- 生成 `.run/godot/pet_growth_observation_100.csv` 供表格分析。

`--auto-pet-rebirth-mm-check` 重点覆盖：

- MM 石头购买和喂养。
- 1 转 MM 转生目标宠。
- 2 转 MM 一次性领取。
- MM 蛋使用。
- 任务引导 marker 和重复任务 marker。

## 边界

- 当前不是最终数值平衡。
- 当前没有正式宠物美术资产，只做 UI 和规则闭环。
- 当前成长阈值仍在模型内，后续正式数值策划应迁移到数据表。
- 当前综合评价使用统一战力成长口径，不按宠物定位加权。
