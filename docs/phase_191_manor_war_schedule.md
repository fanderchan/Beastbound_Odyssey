# Phase191：庄园战期与结算流程

本阶段继续推进九大庄园，不复刻 StoneAge 8.0 源码，但参考其庄园战状态机：
原版存在挑战、准备、开战、休战、结束等阶段；Beastbound 先落地服务器权威的两段式流程。

## 已实现

- 族长在家族面板对庄园宣战，服务端创建 `manorWars` 战期记录。
- 同一庄园同一时间只允许一场进行中的庄园战；同一家族同一时间只能参与一场庄园战。
- 客户端庄园行显示当前战期双方，参战族长可点击 `开战` 结算。
- 结算后生成 `manorBattles` 战报；挑战方胜利才会写入庄园占领状态并解锁对应庄园道具场。
- MySQL 展开表新增 `manor_wars`，`ops status` 统计新增 `manor_wars`。

## 暂不做

- 真实定时赛程、跨天准备期、报名排名表。
- 多人进场和庄园战专用 battle room。
- 庄园地图、庄园 NPC、入场传送、休战期保护。
- 家族银行、税收、守护兽和长老权限。

## 设计备注

- 当前 `startsAt` 立即可结算，是为了让 PC 版可玩链路不中断；后续接入真实排程时只需把准备时间从 0 调整为配置值，并让服务器维护战期推进。
- 商店权限仍然只看最终庄园占领状态，不看宣战记录，避免未结算战期提前解锁道具场。
