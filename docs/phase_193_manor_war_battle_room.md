# Phase193：庄园战专用战斗房间

本阶段把庄园战从“报名名单 + 战力结算”推进到真实联网 battle room。

## 参考结论

- StoneAge 8.0 的庄园系统有 `FMPOINT` 占领结构、庄园 PK 列表与挑战排程。
- Beastbound 不复制原版源码或资源；本阶段只沿用“占领状态 + 挑战排程 + 家族对战结果写回”的结构意图。

## 已实现

- 新增 `manor_war` battle room 模式。
- 已占领庄园被挑战时，挑战方和防守方都报名后，族长可在家族面板点击 `入场` 开启庄园战房间。
- 庄园战房间从参战名单生成双方玩家与出战宠 actor，并保留 N vs N actor ID 唯一性。
- `manor_war` 胜负按整方玩家是否全部倒下判定，不再沿用切磋“一名玩家倒下即结束”的规则。
- 庄园战房间关闭时会生成庄园战报，并按胜方阵营写回庄园占领状态。
- 客户端庄园战结果使用“庄园战”文案，不再显示普通切磋结算文案。

## 当前边界

- 中立庄园守备队仍使用原有 `结算` 路径，不创建 NPC 守备 battle room。
- 真实定时赛程、战前入场地图、庄园 NPC、观战、踢人、锁名单、休战期保护尚未接入。
- 战斗房间仍是运行时态；重启后不恢复 battle room，但 `manorWars` 与庄园战报会随服务器状态文档持久化。
