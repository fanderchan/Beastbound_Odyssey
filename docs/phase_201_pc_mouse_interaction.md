# Phase201 PC 鼠标交互精简版

本阶段确认 PC 端鼠标交互方向：不用右键也必须能完整游玩。右键不做大而全的隐藏功能系统，只保留少量辅助用途，让主流程继续依赖左键、双击、拖拽和明确按钮。

## 最终交互结论

### 主操作

- 左键：移动、选择、交互、打开面板、提交按钮。
- 双击：快速执行，例如使用物品、装备、存入或取出 1 个。
- 拖拽：背包、银行、装备、快捷栏之间移动物品。
- 右键：不承担主流程，只做少量辅助。

### 右键保留范围

只保留三类右键：

1. 右键空地或玩家：只改变人物朝向，不移动。
2. 右键物品格：打开物品辅助菜单。
3. 右键战斗目标：查看详情，不提交指令。

其他 NPC、底部栏、任务、地图、商店普通商品、邮箱普通区域都不做右键主功能。

## 右键朝向规则

### 右键空地

- 不移动。
- 不寻路。
- 不取消寻路。
- 不弹地面菜单。
- 只根据玩家当前位置到点击位置的方向，更新人物朝向。
- 朝向按 8 个方向吸附：上、右上、右、右下、下、左下、左、左上。
- 如果点在自己脚下或方向无法计算，则保持原朝向。

### 右键其他玩家

- 不移动到对方身边。
- 不打开玩家交互菜单。
- 只让自己朝向该玩家。
- 玩家互动仍然用左键打开面板。

这样右键有一个明确用途：方便玩家主动摆姿势、面对镜头、面对玩家或面对场景，给以后拍照、截图、表情动作、社交站位留空间。

## 各界面设计

### 世界地图

- 左键地面：移动。
- 左键 NPC：靠近并交互。
- 左键玩家：打开玩家互动面板。
- 右键地面：改变朝向。
- 右键玩家：朝向该玩家。
- 右键 NPC：暂不做特殊行为，避免左键和右键两套交互造成困惑。

### 背包 / 银行物品格

- 左键物品：选中。
- 双击物品：执行默认动作。
- 拖拽物品：移动、交换、合并、存入、取出。
- 右键物品：打开小菜单。
- 菜单只放辅助项：查看详情、使用或装备、存入或取出、拆分数量、丢弃。
- 丢弃必须二次确认。

### 战斗

- 左键指令。
- 左键目标。
- 右键目标：查看详情。
- 右键绝不提交攻击、捕捉、逃跑、防御等指令。
- 逃跑仍然按敏捷进入战斗出手队列。

### 商店 / 交易所 / 邮箱

- 主流程全部靠左键、输入数量、明确按钮。
- 不把购买、出售、领取附件设计成右键必需功能。
- 如物品以格子形式展示，可以复用“物品格右键查看详情”，但不能成为唯一入口。

## 实现注意事项

- 世界右键必须走独立的“更新朝向”路径，不能复用左键移动路径。
- 右键朝向更新后，需要同步在线位置里的 `facing`，让其他玩家也能看到朝向变化。
- 朝向同步不能高频刷请求；只在朝向实际变化时发送一次位置同步。
- 如果正在自动寻路，右键改朝向不应清掉当前寻路目标，除非后续另行设计“手动打断寻路”。
- UI 区域右键不应穿透到地图，避免点窗口时改变人物朝向。
- 物品右键菜单和拖拽预览必须在窗口上层显示，不被银行或背包面板遮挡。

## 验收标准

- 不用右键也能完整完成移动、任务、背包、银行、商店、战斗。
- 右键空地只改变朝向，角色不移动。
- 右键其他玩家只朝向对方，不移动、不弹玩家菜单。
- 左键其他玩家仍然能打开玩家互动面板。
- 右键物品格能打开辅助菜单。
- 右键战斗目标只查看详情，不提交战斗指令。
- 右键 UI 面板区域不会穿透到地图。
- 朝向变化会被在线位置同步，双开窗口可看到对方朝向变化。
- 跑 `--auto-facing-check`，并补一个右键朝向自动检查。
- 跑 idle / moving `--perf-probe`，确认没有输入或 HUD 热路径回归。
