# Phase204：绑定资产与战斗宠教学补领

## 参考意图与 Beastbound 规则

- 采用 MMORPG 常见的“绑定 / 非绑定”所有权语义，但不复制外部项目的数据或实现。
- Beastbound 中“绑定”只表示不能把所有权转给其他玩家，不等于物品或宠物被锁死。
- 绑定物品仍可使用、丢弃、存入同角色银行，并可由系统邮件发放；绑定宠物仍可切换状态、存入兽栏、培养、改名或清理。
- 非绑定是旧数据和未配置资产的兼容默认值。

## 数据与权威契约

- 物品目录使用 `binding: "bound" | "unbound"`。背包槽仍只保存 `itemId/count`，不引入会被现有堆叠规范化丢失的槽级绑定字段。
- 宠物实例使用 `petInstances[].binding`，旧实例缺省为 `unbound`。
- 宠物蛋孵化时，生成的宠物实例继承来源蛋的 `binding`；本地模型和 Node 权威实现保持一致。
- `novice_battle_pet_egg` 本阶段配置为 `bound`，由它孵出的四灵幼兽自动为 `bound`。
- 服务端在交易所上架和购买、玩家邮件附件、面对面交易发起与接受时校验绑定状态；客户端列表过滤只是易用性，不是授权边界。
- 绑定宠物不能通过 `pet_drop` 放到 `pickupMode=public` 的公共区域。当前没有宠物交易所、宠物邮件或宠物面对面交易接口。

## 教学补领闭环

- “设置战斗宠物”任务仍在进行时，若背包有蛋则打开背包；若蛋在银行则打开银行；若四灵幼兽在队伍则打开宠物界面，若在兽栏则先引导到阿牧并预选该宠。
- 若背包、银行、队伍和兽栏都没有该蛋或四灵幼兽，任务路线指向兽栏管理员阿牧，并显示“补领对战宠物蛋”。
- 补领是服务端权威档案动作；服务端再次检查任务状态、背包、银行、目标宠和背包空间，成功时只加入一枚。
- 背包已满时补领按钮仍显示，点击后明确提示先整理背包，不让玩家到达阿牧后失去操作入口。
- 因为绑定蛋不能跨账号转移，且银行内已有蛋时不能补领，所以允许丢弃后补领不会形成重复复制路径。

## 非目标

- 本阶段不实现拾取绑定、装备后绑定、账号绑定、槽级混合堆叠或宠物交易。
- 不回填旧四灵幼兽的来源；没有可靠来源记录的旧宠按兼容默认值处理。
- 不改变银行、NPC 商店、任务交付、合成消耗和系统邮件领取的既有规则。
- 这是向后兼容的数据与校验扩展，不提升客户端协议版本。
