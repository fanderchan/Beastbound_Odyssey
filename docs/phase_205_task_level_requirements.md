# Phase205：全任务接取等级与推荐等级

## 玩家规则

- 每个正式任务都必须显式配置顶层字段 `requiredLevel` 与 `recommendedLevel`；目录校验会拒绝漏配，而不是依赖隐藏默认值。
- 所有玩家可见任务标题统一显示为 `[接取等级] 任务名`，Lv1 任务也显示 `[1]`，避免同一任务系统出现两套视觉规则。
- `requiredLevel` 只负责接取门槛：未达到等级时不能新接任务，也不会跳过它去激活后续主线；任务一旦接取，即使人物之后因转生等原因降级，仍可继续推进、交付和领取。
- `recommendedLevel` 只负责说明提示，不参与硬校验。任务面板和 NPC 任务介绍固定显示 `接取等级：LvN` 与 `推荐等级：LvN`。

## 当前内容分级

- `quests.json` 共 44 个正式任务，全部已有两种等级字段。
- 现有新手主线、功能教学和支线共 38 个，实际发生在 Lv1 野怪与新手设施内容中，统一为接取 Lv1、推荐 Lv1。
- 一至六转资格任务共 6 个，依据现有人物转生规则设为接取 Lv80；依据元素洞试炼现有推荐值设为推荐 Lv100。
- 宠物转生 MM 教学保持接取 Lv80、推荐 Lv130；它是独立状态机，不混入 `quests.json` 的人物转生资格任务。

## 宠物转生教学

- 独立的宠物转生 MM 教学不走 `quests.json`，本阶段为它补齐同一份等级展示契约：最低接取等级 Lv80，推荐等级 Lv130。
- 截图对应步骤现在显示为 `[80] 宠物转生教学：领取1转小MM`，详情固定包含 `接取等级：Lv80` 与 `推荐等级：Lv130`。
- Lv79 及以下时，阿澄头顶显示锁定任务标记，对话按钮显示 `需要 Lv80` 且不可点击；达到 Lv80 后才恢复“开始教学”。
- 本地档案与 Node 服务端都会拒绝低等级接取。服务端战斗结算还会复核等级和教学状态，伪造 `pet_rebirth_mm_trial_1` 遇敌也不会发放1转小MM。
- 教学进行中已经持有1转小MM时，重复伪造同一试炼不会再发一只；先继续喂石、练级并完成转生。教学完成后的循环挑战仍可每次领取，但人物当前等级必须重新达到 Lv80。
- 推荐 Lv130 不是硬门槛；Lv80 可以接取和挑战。已完成教学不会被重复的开始请求改回进行中。

## 普通任务顺序与兼容

- 普通任务等级判断集中在 `QuestModel` / `PlayerProgressModel` / Node 任务模型，不把规则散落在任务面板或 `main.gd`。
- 等级不足的首个未完成主线会停在原位；达到等级后再自动成为当前任务，不扫描到后续无等级任务。
- 只有真实的 `requiredLevel` 等级锁会占用任务面板/追踪 fallback；未来二转、三转等阶段锁不会遮住正在进行的宠物转生 MM 教学。
- 服务端不再在“当前没有可用任务”时回退激活第一条已完成的新手任务。
- 领取前序任务时，如果下一任务尚未达到接取等级，不提前创建下一任务状态；达到等级并正式激活后，任务状态不再受人物当前等级影响。
- 字段仍是数据层扩展，不改变 HTTP/WS 结构，不提升协议版本。
