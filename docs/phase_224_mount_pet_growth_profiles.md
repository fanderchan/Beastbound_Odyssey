# Phase 224：新手老虎与雷龙坐骑成长接档

日期：2026-07-11

## 范围

P0.2d-2c 为最后两个未接档形态接入服务端权威成长：

- 新手老虎：强制教学免费发放的绑定高速骑宠；
- 雷龙：钻石商店或玄影庄园商店获取的高血防骑宠。

本阶段不改变蛋的来源、价格、绑定、骑乘公式、技能、视觉或现有通用成长中心。目标是把新获得坐骑纳入与战斗宠相同的 authority-v1 私有个体、逐级成长、公开观察和 EXP 结算闭环，而不是借接档重做商业宠经济。

临时 Pet Design Contract 位于 `.run/pet-design/p0_2d_mount_growth_backfill.json`，已通过项目宠物设计 Skill 校验，不提交版本库。

## 设计判断

### 新手老虎

- 保留 Lv1 基础：生命 48、攻击 8、防御 5、敏捷 28。
- 保留每级中心：生命 7.3、攻击 1.35、防御 0.95、敏捷 1.95。
- 定位是早期高速骑乘与骑宠教学，速度明显、耐久与综合终局偏低；只有攻击/防御基础指令，不替代正式战斗宠。
- 免费教学价值来自立即获得骑乘能力与可培养反馈，不来自终局数值碾压。

### 雷龙

- 保留 Lv1 基础：生命 92、攻击 12、防御 18、敏捷 12。
- 保留每级中心：生命 11.5、攻击 1.35、防御 1.90、敏捷 0.72。
- 定位是厚重护卫骑宠；付费/庄园价值来自显著血防、骑宠承伤和稀有来源，代价是低敏低攻。
- 3600 钻石价格属于后续 P1.6 商业经济审计范围。本阶段不在没有 PvE/PvP 实战证据时偷偷提高或降低付费宠中心数值。

两档均使用 `weighted_center + 2% extreme` 个体分布，保留 Lv1 4V 与隐藏成长的随机观察乐趣。

## 万人模拟

每档使用正式 Node authority 算法模拟 10,000 个个体，从 Lv1 逐级结算到 Lv140，并写入 2–140 共 139 级观察阈值：

| 形态 | 三项成长/级范围 | Lv140 战力范围 | 平均 |
| --- | ---: | ---: | ---: |
| 新手老虎 | 3.755–4.719 | 827–976 | 897.64 |
| 雷龙 | 3.540–4.353 | 941–1091 | 1016.27 |

层级保持清楚：新手老虎略高于芽耳布伊的约 858，但低于普通长期主力；雷龙高于免费坐骑，却仍明显低于五转四灵幼兽的约 1225。战力公式没有完全表达骑宠替人物吸收伤害的防守价值，因此雷龙不能只按这一列判断价格合理性。

## 权威与旧档兼容

- 两个模板分别连接 `novice_tiger_mount_v1` 与 `thunder_dragon_mount_v1`。
- 新孵化、GM 创建和后续新奖励实例由 `createNewPetFactory` 直接生成 canonical authority-v1，公开响应不含 private seed。
- 骑宠通过共享 EXP dispatcher 逐级结算；骑乘状态、被击落休息和人物承伤公式不变。
- 已存在但没有 authority envelope 的同形态坐骑继续解析为 `legacy_existing`，只保留旧等级/经验行为，不补种子、不改四维、不重抽价值。
- 生产目录现为 `32/32` 形态全部接档；`legacy_unlinked` 兼容路径改用注入的旧格式测试夹具保留覆盖，不再依赖某个正式宠物永远不接档。
