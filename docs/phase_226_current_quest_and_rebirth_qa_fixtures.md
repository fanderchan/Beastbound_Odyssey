# Phase 226：当前任务链与转生 QA 夹具收口

日期：2026-07-11

## 范围

P0.2d 完成后的扩展回归复现三个过时 Godot QA 假设：

1. 人物转生执行检查仍假定“捕捉乌力”之后立即进入一转任务；
2. 宠物安全检查把任意已有乌力当成“捕捉中毒乌力”任务的不可动资产；
3. 六转链检查假定人物每次转生回到 Lv1 后会立即接到下一转 Lv80 任务，远程兽栏检查又依赖默认档案自带宠物。

本阶段只修 QA 夹具与测试辅助函数，不改变任务数据、宠物保护规则、转生规则、玩家默认档案或服务器行为。

## 当前产品语义

### 新手任务链

正式顺序为：

`捕捉乌力 → 打开图鉴 → 打开家族 → 打开账号 → 一转资格`

因此转生执行检查同时锁定 `quest_capture_wuli → quest_open_codex_panel` 和 `quest_open_account_panel → quest_rebirth_1_guidance`，不再跳过中间教学。

### 捕捉任务与已有宠物

“捕捉乌力”要求使用缚毒捕捉网捕捉处于中毒状态的新乌力。玩家已经拥有的任意乌力不是该任务要交付的资产，因此可以休息、待机、培养或从兽栏清理。锁定宠、骑乘宠和真正匹配交付型目标的保护逻辑没有放宽。

安全检查现在验证：

- 锁定宠仍不能丢弃、交付、批量改动或作为培养材料；
- 骑乘宠仍跳过批量状态切换；
- 捕捉任务不会错误冻结玩家已有同系宠物；
- 批量休息/待机的 changed/skipped 数量与当前规则一致。

### 连续转生

人物完成一转后回到 Lv1，不应立即显示要求 Lv80 的二转资格。QA 辅助流程现在明确模拟“重新练到 Lv80”，随后才验证下一转任务出现。六转完成后仍无后续任务，并继续验证六转上限。

`_profile_after_rebirths_for_test()` 同步执行这一测试过程，因此三转后的远程兽栏检查会先达到 Lv80、正确出现四转资格。远程兽栏 UI 夹具显式创建 `pet_bui_tough`，不再依赖已经为空的新账号默认宠物列表。
