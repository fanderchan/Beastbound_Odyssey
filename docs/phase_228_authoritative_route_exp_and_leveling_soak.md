# Phase 228：权威路线经验、最后一击参与结算与 Lv1→140 隔离 Soak

日期：2026-07-11

## 初始问题

Phase227 已让正式地图真实覆盖 Lv1–140，但经验仍有四个发行级断点：

1. Godot 数值报告读取 `reward_economy.json` 的等级段倍率，Node 正式战斗只用怪物四维基础值，两端不是同一事实；
2. 旧 `groupMultipliers` 除资格战外全为 1，11 个区域只有 5 个经验目标命中，9 个重复练级区只有 4 个战数目标命中；
3. 雾帽、裂日、风镜每张地图各有前后两个等级池，正式路线却只引用其中一个，边界等级可能被带去刷过低或过高的怪；
4. 经验归属必须保留石器式参与规则：只有造成最后一击的行动者，或实际参与该次最后一击合击的行动者获得经验，不能因为人物和战宠属于同一账号就自动共享。

## 版本化经验口径

保留历史 `battle_exp_v1` 与 `phase123_core_v1`，新增并启用：

- balance set：`p0_3_route_v2`；
- EXP formula：`battle_exp_v2_route`。

基础四维和等级缩放公式不变，只按既有 `progression_zones` 目标范围校准阶段倍率：

| 奖励档 | 倍率 | 目的 |
| --- | ---: | --- |
| 新手/default | 1.0 | 保持教学速度 |
| `growth_training_01` | 1.8 | 雾帽与裂日命中初期目标 |
| `rebirth_prep_training_01` | 1.25 | 风镜与四洞一层命中一转准备目标 |
| `post_rebirth_training_01` | 1.5 | 转后段不再显著低于目标 |
| `high_cave_training_01` | 2.2 | 四洞深层保持 30–75 战/级目标 |
| `shadow_chase_training_01` | 4.5 | 玄影追赶段命中 28–55 战/级目标 |
| `shadow_capstone_training_01` | 6.0 | Lv132–140 命中 30–65 战/级目标 |

这不是改怪物属性或捕捉率；倍率只影响新发生的战斗经验。旧战报继续保留旧 balance 摘要，旧档案、宠物实例和已获得经验不迁移、不重算。

## 服务端事实统一

新增严格 `battle-exp-catalog.js`：

- 固定读取共享 `reward_economy.json` active formula；
- 拒绝缺 active、数字字符串、负倍率、缺 `default_wild` 等配置；
- Node 与 Godot 使用 7 个固定样本锁定完全相同的取整结果；
- 正式权威遇敌无条件按服务器选定的 form、等级、四维和 `rewardTableId` 生成 `expReward`，地图里的旧显式经验不能旁路统一公式；
- 客户端伪造 form/等级/四维/经验仍被忽略。

等级差规则继续为：高于敌人不超过 5 级拿全额，之后 15 级线性衰减，最终最低 1 点。Node 正式结算现在直接复用该纯函数。

新增严格 `player-level-runtime.js`，固定读取 `level_curves.json`，正式人物升级和 soak 共用同一 `expToNextLevel` 与跨级/满级溢出算法。Lv1、20、60、80、100、120、131、139、140 等锚点有固定测试。

## 最后一击参与者结算

PVE 击杀经验继续只发给最后一击的实际参与者：

- 人物单独击杀：人物获得经验，其出战宠不共享；
- 战宠单独击杀：战宠获得经验，人物不共享；
- 人物和战宠实际参与最后一击合击：两名参与者各自获得经验；
- 人物获得经验时，骑宠继续按人物经验的 60% 获得经验。

同账号归属不是参与证明。待机宠、没有加入最后一击的出战宠、其他未参与击杀的队员和倒下宠物都不会搭便车。训练伙伴人物与其宠物快照也分别按实际最后一击参与情况结算。

这项规则只改变经验归属，不复制石币、物品、捕捉结果或任务击杀次数。多人队伍的跨账号经验分配没有在本阶段扩大。

## 早中期路线细分

将三个大段细分为真实前后池：

- 雾帽芦苇 Lv10–17 / 镜池 Lv18–24；
- 裂日灌木 Lv25–34 / 岩脊 Lv35–44；
- 风镜草坡 Lv45–54 / 断崖 Lv55–64；
- Lv65 起进入四洞一层。

正式路线因此从 9 段细化为 12 个重复练级段，所有分组、等级和奖励档均来自真实地图。

## 隔离新号 Lv1→140 Soak

`tools/progression_leveling_soak.mjs` 每次创建全新的 memory 账号，从真实默认 `player.level=1 / exp=0` 开始，然后逐战调用：

1. 严格正式路线选择；
2. 服务端权威地图遇敌与宠物池抽取；
3. `battle_exp_v2_route`；
4. 正式等级差衰减；
5. 正式 `player-level-runtime`。

固定种子 `p0_3_leveling_soak_v1` 结果：

- Lv1→140：5,210 战；
- Lv20：231 战；Lv65：1,059 战；Lv80：1,638 战；Lv100：2,769 战；Lv120：4,170 战；Lv132：4,808 战；
- 中高阶路线遇到 37 次稀有 Lv1 条目；
- 12/12 阶段平均战数全部落入各自目标范围；
- 5 个补充固定种子总战数 5,200–5,244，稀有 Lv1 为 33–43 次，均无越界阶段。

单个等级允许因剩余经验、等级差和 1% Lv1 惊喜产生波动；验收使用阶段平均战数，因为 `targetBattlesPerLevel` 本来就是阶段目标，不是强制每一级完全相同。

## 真实在线挂机证据

隔离服务创建 Lv70 账号并在岩脉洞穴一层开启 `walk` 挂机会话，服务端逐格移动签发一次性许可，真实创建 `rebirth_prep_training_01` 战斗并结算：

- 人物或战宠中只有实际完成最后一击的一方获得权威经验，另一方保持 0；
- 奖励表保持 `rebirth_prep_training_01`；
- `hangSession.battleCount` 从 0 增至 1，挂机保持启用；
- 未使用客户端怪物或经验字段。
