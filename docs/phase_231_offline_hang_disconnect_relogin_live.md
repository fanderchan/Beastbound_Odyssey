# Phase 231：离线挂机真实断线、重登录与领取闭环

日期：2026-07-11

## 隔离原则

这项验证没有使用本机 8787 后端、真实 MySQL、auth1373 或任何现有玩家档案。测试在 8799 启动一次性 memory 后端，结束后确认端口关闭，并保留原 8787 进程不动。

隔离服务器可显式注入 `qaAdvanceClock`；只有注入时才开放 `POST /__qa/clock/advance`，正式 `createHttpServer()` 默认返回 404。这样可以验证真实 HTTP/客户端断线流程，而不等待现实时间，也不把测试时钟变成生产 GM 能力。

## Godot 真实闭环

新增 `--auto-offline-hang-live-check`：

1. Godot 通过真实 HTTP 注册全新服务器账号并拉取空白档案；
2. 加载火芽村入口，把人物移动到 `village_grass` 并同步服务端位置；
3. 通过正式 `ServerSyncCoordinator` 开始离线修行；
4. 原地登出，服务端 session 被注销但 profile 中 active 离线会话保留；
5. 隔离时钟前进 6 分钟；
6. 同一账号重新登录，档案恢复 active 会话且客户端产生一次领取提示；
7. 通过正式客户端动作领取，权威档案 revision、等级、石币与账本同步回客户端。

固定结果：50% 默认比例、30 秒/场基准，6 分钟折算 6 场；获得人物经验 187、石币 108，人物从 Lv1 升到 Lv2，账本 1 条。
