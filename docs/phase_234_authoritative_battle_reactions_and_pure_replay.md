# Phase 234：服务端权威战斗反应与 Godot 纯回放

日期：2026-07-11

## 先复现的问题

Phase 233 只建立了私有战斗随机和反应纯规则，尚未在联网回合真正启用。协议 v3 的 Godot 映射会把服务端 `damage=0` 夹成至少 1 点伤害，不认识独立 `counter_attack`，并继续用客户端状态重掷反应；如果只先发布服务端，玩家会看到重复伤害、丢反击或最终快照突然纠正的错误表现。

审查还复现了两个契约问题：

- 新增 `dodged/critical/counterTriggered`、零伤害闪避和独立反击后，事件列表与伤害事件仍标 `schemaVersion=1`；
- 既有 10v10 自动检查仍用“新账号空宠档”却断言战宠出手，因此编队只有九名我方单位。这是显式 QA 夹具缺失，不是战斗模型容量不足。

## 玩家可感知合同

- 服务端决定闪避、幸运一击、最终伤害、最终 HP 和是否反击；客户端只播放结果。
- 普攻可闪避、可幸运一击、可触发反击；伤害宠技可闪避和幸运一击，但不会触发反击；合击三项全部关闭。
- 闪避后的存活目标仍可按现有石器式手感触发反击；守护、石化、睡眠及击杀会按既有规则阻断对应反应。
- 反击是紧跟来源攻击的独立事件，带 `counterSourceEventId`，自身可被闪避或触发幸运一击，但绝不会产生第二层反击。
- 本阶段复刻当前已经运行的 legacy 概率和取整顺序，不借“权威化”暗改闪避率、暴击率、暴击伤害或反击伤害。

## 服务端权威结算

新增 focused `battle-reaction-resolver.js`，只接受 Phase 233 的私有随机 authority：

- `dodge.v1`、`critical.v1`、`counter.v1` 的原始 roll、secret、purpose 和概率均不进入事件、trace 或存档；
- `basic_attack`、`pet_skill`、`combo_attack`、`counter_attack` 的伤害事件输出最终 `damage/hpBefore/hpAfter/dodged/critical/counterTriggered`；
- 反击先用服务端普通攻击公式取得底伤，再应用既有 75% 反击系数，随后独立结算闪避和幸运一击；
- 反击击杀继续走同一 PVE 最后一击经验、击飞和宠物 HP 写回路径，不另造奖励通道。

战斗事件 envelope 和四类伤害事件统一升级为 `schemaVersion=2`。嵌套参与者、经验条目及未改变语义的旧事件继续保持自身 v1，避免伪造无意义的全量 schema 升级。

## Godot 纯回放

`server_battle_room_model.gd` 将四类伤害统一映射成 `serverResolved` 本地事件：

- 接受合法零伤害，不重选目标、不重掷闪避/幸运一击、不再次乘伤害、不生成本地反击；
- 精确写入服务端 `hpAfter`、击飞、战败和反应事实；
- 敌我双方状态技能都使用服务端精确目标与强制命中结果；
- 反击来源 ID、服务端事件 ID 和宠物反击 `skillId` 进入事件 ledger，便于审计回放顺序。

断线重连收到“已经结算的最终 actor 快照”时，回放入口会先按每个目标第一次出现的 `hpBefore` 恢复回合起点，并清除与活体矛盾的击飞/倒地标记。逐事件播放结束后再应用最终 actor 快照，HP、骑宠可见事实、状态和击飞事实均不需要二次纠正。

`main.gd` 只在 legacy 本地事件中插入 `lastCounterEvent`；服务端反击必须作为独立事件到达，杜绝双反击。

## 协议部署边界

客户端和服务端协议从 3 同步升级为 4，兼容窗口固定为 `4..4`。协议 3 和未来协议均收到 HTTP 426 与中文升级提示。服务端反应事件、Godot 纯回放和协议 4 必须作为同一个部署组发布，不允许拆开灰度。

这次不改变存档 schema、HTTP mutation 请求、宠物成长、捕捉、转生或经济规则。

## 已发现但不在本片掩盖的缺口

1. P0.4d：联网中毒尚无权威 `status_tick`，睡眠受伤解除与状态回合递减也未完成。
2. P0.4e：服务端目前不执行骑宠 50% 分伤，也不回写骑宠 HP。当前 Godot 的 `actorDamage=damage / rideDamage=0` 与服务器事实一致，但不符合既有骑乘战斗承诺；后续必须由服务端输出人物/骑宠分伤及双方 HP，再由客户端纯回放，不能恢复客户端假分伤。

## 后续玩家手感验收

状态生命周期和骑宠分伤完成后，再用两个 QA 账号连续进行至少十回合切磋：

1. 观察“攻击 → 闪避 → 反击”是否严格按日志顺序播放，且反击最多一次；
2. 确认幸运一击只出现一次伤害浮字，HP 不在回合末跳第二次；
3. 确认伤害宠技被闪避时不产生反击，合击不触发三类反应；
4. 在反击后立即断线重登，确认回放与最终 HP、击飞状态完全一致。

通过标准：无幽灵反击、无重复扣血、无回合末 HP 跳变、无服务端日志和客户端表现次序不一致。
