# Phase 235：联网异常状态生命周期与纯回放

日期：2026-07-12

## 先复现的问题

Phase 234 已让闪避、幸运一击和反击成为服务端事实，但联网异常状态只完成了施加、免疫、抵抗和净化，没有完成整个回合生命周期：

- 毒物命中后，目标从 168 降到 156，并得到 `poison turns=3 potency=6`；下一整回合却没有 `status_tick`，HP 和持续回合均不变化。
- 睡眠目标受到 21 点普通攻击后仍保留睡眠，随后又产生 `status_skip 2→1`，与既有“正伤害唤醒睡眠”规则不一致。
- 睡眠和石化已经按被控者自己的行动机会递减：高速施加会在同回合消耗第一次，低速施加则从下回合开始消耗。这个外部手感正确，本阶段予以保留。
- Godot 的联网回合会显式关闭本地回合末毒伤；因此服务端不输出 tick 时，客户端不能安全补算。
- 重连拿到的是回合终态，原回放入口只按首个 `hpBefore` 回卷 HP，不恢复异常状态、骑宠和行动事实，会造成少播、重复递减或最终快照突然纠正。

## 参考意图与 Beastbound 取舍

本地 StoneAge 8.0 参考只用于确认行为意图：睡眠/石化按单位行动机会消费，正伤害会唤醒睡眠，中毒独立于普通攻击反应链。参考代码并非官方规格，也不复制其源码或数值。

旧普通毒存在保留 1 HP 的实现，但 Beastbound 既有离线模型和本路线图已经明确中毒可致死。本阶段保持 Beastbound 规则，不把权威化变成暗中削弱毒伤。

## 玩家可感知合同

- 中毒在正常命令全部结算后、按稳定 actor 顺序每回合跳伤一次；新施加的毒在施加回合末立即跳第一次。
- 毒伤使用施加时冻结的强度，随后严格 `3→2→1→0`；可以致死，并进入正常胜负、经验和 HP 回写。
- 毒伤不闪避、不触发幸运一击、不反击、不击飞，也不唤醒睡眠；毒物和毒精灵的即时伤害同样不唤醒。
- 净化若先于回合末发生，本回合不再补毒伤；免疫或抵抗从不创建持续状态。
- 睡眠和石化在该 actor 尝试行动时生成一次权威 `status_skip` 并消费一次持续回合。
- 普攻、伤害宠技、合击或反击造成真实正伤害时解除睡眠；闪避、零伤害和毒伤不解除。若目标在自身行动前被打醒，本轮已提交的命令可执行；若已经跳过，之后醒来不会补行动。
- 石化不会因受伤解除。本阶段只补生命周期，不顺手实现石化防御加成。
- 若某个稳定顺序中的毒伤已形成战斗结果，后续毒伤停止，暂不新增同时 DOT 双杀或平局规则。

## 服务端权威实现

`battle-status-rules.js` 集中承担私有状态随机、施加者归属冻结和公开状态投影：

- 状态命中改用房间私有随机 authority，命名空间为 `status.v1`，上下文包含 `statusId`；公开事件只给最终命中事实，不给原始随机值或 secret。
- 状态成功施加时冻结实际 actor 的人物、战宠、陪练伙伴和当时骑宠归属字段；公开 actor、事件和重连快照只投影 `id/label/turns/potency/sourceId`。
- 施毒者之后即使换宠、死亡或离场，最后一击仍归原施毒单位。人物施毒时人物与当时骑宠按既有 60% 规则结算；战宠施毒只归该战宠，符合“未参与最后一击不拿经验”。

回合结算新增稳定的状态阶段：

1. 清除上回合瞬时守护状态；
2. 捕获公开 `actorsBefore`；
3. 按速度执行命令、处理睡眠唤醒与睡眠/石化跳过；
4. 若尚未结束，按 actor 数组稳定顺序结算中毒；
5. 每次毒伤后检查结果，最后生成终态 `actors` 并执行原有结算和写回。

`status_tick` 和 `status_skip` 均为 schema v2，携带 `hpBefore/hpAfter`、`statusBefore/statusAfter`、`fromTurns/toTurns`、`statusChanges`、来源与最终反应事实。四类直接伤害事件在唤醒睡眠时附加 `remove_on_damage`，且在反击判定前完成状态移除。

`lastEventList` 保留前后 actor 快照供当前回合重连；最多 20 actor 的测量约从 31.3 KiB 增至 52.0 KiB。历史 `eventLog` 只保留元数据和 events，不再把两份全量快照复制进最近 20 回合，避免约 1 MiB/房间的持续放大。

状态仍只存在于运行时 battle room，不写入玩家档案，也不改变旧存档 schema。

## Godot 纯回放与重连

Godot 收到事件列表后先从 `actorsBefore` 构造临时回合起点，再基于该起点映射事件；只有映射成功才替换正在显示的战斗状态。没有 `actorsBefore` 的旧夹具继续使用首个 `hpBefore` 回退。

服务端状态事件统一标记 `serverResolved`：

- `status_tick/status_skip` 直接写服务端 HP 和状态终值，不再次扣血或 decrement；
- 毒物、毒精灵、状态技能和净化按每个服务端目标事实回放，不重掷命中；
- 敌方使用物品不会消耗本地背包，本地人物使用物品则按服务端 `remainingItemCount` 精确同步；
- 直接伤害是否解除睡眠只服从服务端 `statusChanges`；
- event ledger 保留服务端事件 ID、来源、HP 与状态前后证据。

逐事件回放完成后再应用终态 `actors`；聚焦检查确认 HP、异常状态、倒地/击飞、骑宠可见事实和权威身份均不发生第二次变化。

## 协议部署边界

协议从 4 同步升级为 5，客户端和服务端兼容窗口固定为 `5..5`。协议 4 不认识权威 `status_tick`，若继续放行会漏动画并在终态快照突然掉血，因此旧客户端收到 HTTP 426 与中文更新提示。

协议 5 客户端、服务端状态结算和纯回放必须作为同一个部署组发布。本次没有改变存档、HTTP mutation 输入、宠物成长、状态数值或经济规则。

## 非目标与已登记缺口

- P0.4e：联网骑宠分伤、倒下与 HP 回写。
- P0.4f：混乱重定向、混乱持续消费和石化防御效果。
- 不决定换下宠再次换回时是否保留异常状态。
- 不让异常跨战斗或写入玩家档案。
- 不修改命中率、抗性、免疫、持续时间、毒伤强度、技能目录、美术或音效。
