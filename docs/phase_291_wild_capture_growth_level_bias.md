# Phase 291：全物种捕捉等级成长长尾与战斗/捕后同个体

## 目标与复现

本切片解决两个同源问题：

1. 旧 `partyPveEnemyActors` 先从地图 `battleStats` 生成战斗野怪，`pet-capture-candidate-authority` 再单独生成捕获候选。测试可稳定复现战斗 actor 为 `9000/8000/7000/6000`，候选宠却是另一套真实成长数值。
2. 捕获候选的隐藏成长过去与捕获等级无关。Lv1 与 Lv140 都从同一物种分布直接抽一次，导致主动抓高等级宠既省训练时间，又不承担极品率代价。

产品决定是：战斗中的可捕捉野怪与捕获后的宠物必须是同一个体；其真实 Lv1 4V 永远可追溯。Lv1 4V 与隐藏成长继续完全独立，而捕获等级只改变隐藏成长的入选分布。

## 石器 8.0 参考与 Beastbound 差异

本地石器 8.0 参考先在 `gmsv/src/char/enemy.c` 创建完整敌宠对象，捕获时再由 `gmsv/src/char/pet.c` 的 `PET_createPetFromCharaIndex` 从该对象创建宠物，不会在成功瞬间重新抽一只。这支持“战斗与捕后同个体”的玩家心智。

Beastbound 不照搬它的成长关联：

- Lv1 4V 是公开的完美主义追求；
- 隐藏逐级成长是独立随机变量；
- 高等级捕获只压低隐藏成长上尾，不读取或压低 Lv1 4V；
- 高等级仍保留极低的彩票级极品机会。

## 权威个体合同

服务端为每只可捕捉 actor 执行以下顺序：

1. 通过严格新宠 factory 建立真实 Lv1 authority-v1 个体；
2. 仅依据捕获等级和隐藏 `innateGrowthBonus` 选择成长种子；
3. 将选中的同一只宠物结算到野外等级；
4. 在房间公开前，把 form、species、line、level、max HP、攻击、防御、敏捷、元素、主动技能、技能槽和被动技能同步到战斗 actor；
5. 捕获成功后原样 claim/materialize 该冻结候选，不重抽品质。

候选校验同时反查 actor 的固有事实。战斗过程只允许当前 HP 因伤害变化；若 max HP、攻、防、敏、形态、元素或技能与候选不一致，捕捉失败关闭，而不是静默生成另一只。

地图静态 `battleStats` 继续服务于不可捕捉怪、特殊守卫和稳定奖励基准。可捕捉普通野怪的实际战斗固有属性以私有候选为准，因此高等级成长上尾受压也会自然降低练级怪的极端难度。

新捕获的 Lv2+ authority 宠仍保存其真实：

- `initialStats`；
- `growthSpeciesLevel1Stats`；
- `petGrowth.public.levelOneFourV`。

这些字段彼此一致，并继续通过公开档案投影供宠物面板显示；服务端不会用当前高等级属性倒算或伪造 Lv1 历史。

## 等级条件化长尾模型

这不是正态分布，而是“物种基线分布 + 捕获等级条件化的右侧长尾权重”。每个物种仍使用自己的中心、范围和 `uniform/weighted_center/rare_spike` 分布；全局政策只在该物种的隐藏成长范围内计算相对位置。

隐藏成长战力沿用：

```text
growthPower = maxHp / 4 + attack + defense + quick
q = growthPower 在本物种隐藏成长最小/最大之间的归一值，范围 0..1
```

等级压力：

```text
pressure(1) = 0
pressure(L) = (L - 1) / ((L - 1) + 9), L > 1
```

入选权重：

```text
accept(q, L) = 1, q <= 0.5
upper(q) = (q - 0.5) / 0.5
accept(q, L) = 0.0001 ^ (pressure(L) * upper(q) ^ 1.2), q > 0.5
```

含义：

- Lv1 压力为 0，第一抽必收，完全保持物种原始分布；
- 中位及以下不受拒绝，等级越高越集中在中游/下游；
- 只有隐藏成长参与 `q`，`initialBonus`/Lv1 4V 不参与；
- 理论最高成长在任何等级的权重都大于 0，因此彩票尾巴不会消失；
- 每只野宠最多尝试 8 次；若全部拒绝，从已尝试个体中取隐藏成长战力最低者，在线耗时严格有界。

配置集中在 `pet_growth_species_profiles.json#wildCaptureGrowthPolicy`，Node 与 Godot 目录都严格校验字段、范围和正数彩票权重。新增物种只要有成长档，就自动按自身范围应用同一政策，不复制 32 套公式。

## 边界与兼容

- 只影响本版本以后新生成的服务端权威野外捕捉候选。
- 不重抽、不迁移、不削弱任何旧宠或现有玩家数据。
- 不影响奖励宠、蛋、GM 发宠、人物/宠物转生、进化或融合。
- 不改变捕捉成功率、工具消耗、EXP、掉落或协议版本。
- 不向客户端公开 seed、roll、精确隐藏品质、入选次数或兜底事实。
- 捕捉内挂后续只会对真正 Lv1 新捕宠使用公开四项分位；Lv2+ 默认保留。本切片不改该 UI，也不执行自动丢弃。
- 现有 Lv20 成长预览保留为宠物面板人工观察地基，不再作为捕捉专栏自动练宠/自动处置路线。

## 性能

在线路径只在遇敌房间创建时、每只可捕捉 actor 执行一次：

- 期望抽样次数约 `1.25..1.39`；
- 单只硬上限 8 次；10 敌人房间硬上限 80 次；
- 无数据库查询、JSON I/O、客户端计算、每帧逻辑或人口级扫描；
- 选中后沿用既有成长结算和房间私有候选，不增加战斗回合负担。

正式离线审计在本机对 32 档 × 5 等级 × 每档 10,000 只，共 1,600,000 只入选宠物耗时 `31,891ms`，约 `50,171` 只/秒。该数据用于确认算法成本和分布，不是 200 人同图容量声明。
