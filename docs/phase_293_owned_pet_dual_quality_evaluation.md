# Phase 293：宠物面板双独立品质观察与人工去留判断

## 结论

P1.1d-5 已纠正捕捉页与练宠页的职责。捕捉设置只负责寻找目标、投网以及刚捕获 Lv1 宠物的公开四维筛选；原“Lv20 成长保留预览”“待处理”和可能被理解为自动练宠的文案已从捕捉页删除。练级后的成长判断迁回宠物面板，仍是玩家人工决策，不会训练、移动、丢弃或改写宠物。

宠物成长页现在明确并排显示两套互相独立的随机结果：

- `Lv1 4V分位（物种内）`：从该宠不可变的 `initialStats / growthSpeciesLevel1Stats` 计算血、攻、防、敏在同物种 Lv1 分布里的累计百分位。只要具有真实 Lv1 历史，升级和转生后仍可回看。
- `实测成长分位`：只从已经发生的逐级成长计算；Lv1 没有成长样本，Lv2 起可看趋势，Lv5、Lv10、Lv20是人工复核节点。

两图不会合成一个“总品质”，也不会暗示 Lv1 4V 能预测隐藏成长。原成长表的当前值、成长/级、评级和 `预测140` 实测外推均保留。

## 石器 8.0 基线、Beastbound 差异与是否过度设计

本地稳定石器 8.0 参考保留玩家显式查看宠物、显式丢宠、满栏失败及服务端丢宠日志，没有可验证的隐藏成长自动训练/自动删除界面。继续把成长门槛塞在捕捉内挂里，会让玩家误以为系统要代替“抓宠后亲自练级观察”的核心过程，因此不应照旧扩张。

Beastbound 的必要差异是：Lv1 4V 与隐藏逐级成长完全独立，而且玩家需要长期挂机批量培养。额外增加一张 Lv1 物种内分位图能解决真实痛点——宠物练高后仍能确认它当初是否属于四维极品；另一张实测成长图则回答是否值得继续培养。两张简单雷达复用现有成长页和物种目录，没有增加规则树、隐藏评分、自动处置或新容器，收益明确且没有过度设计。

## 数学与公开资料边界

Godot `pet_level_one_percentile_model.gd` 与服务端 `pet-level-one-percentile.js` 使用同一离散 CDF 和 authority-v1 取整规则。它只能读取：

- 物种公开 `outputBase`；
- `initialOutputSpread`、`distribution`、`rareExtremeRate`；
- 宠物公开且两份一致的 Lv1 血、攻、防、敏快照。

它不读取 `growthOutputSpread`、隐藏 seed/roll、内置成长品质、当前高等级四维或未来升级结果。资料缺失、形态与成长档不匹配、两份 Lv1 快照不一致时显示资料不足，不猜测历史。

全物种审计对 32 个成长档各运行 10,000 个确定性 authority-v1 Lv1 样本，共 320,000 只；解析离散 CDF 与真实累计比例的最大绝对误差为 1.64%，低于 2.5% 门禁。

## 人工成长判断与兼容

宠物成长页新增可收起的“人工成长判断”：玩家可设置综合、生命、攻击、防御、敏捷五条 `0..100` 分位参考线，`0` 表示不限。界面根据公开实测成长给出“继续观察”“建议继续培养”或“可考虑放弃”，并逐项解释实际分位与参考线；结果永远只是建议。

联网保存使用新的白名单 action `pet_growth_evaluation_settings_update`。为避免旧存档迁移和旧客户端丢值，策略仍兼容保存在既有 `autoCaptureSettings.growthRulePolicy`；新捕捉页提交时剥离该字段，服务端沿用旧值，新宠物面板 action 独立严格校验并保存。旧客户端仍可走原捕捉设置合同，当前协议版本不变。

所有测试都对宠物数组做前后快照；保存参考线只增加档案 revision，不改变宠物实例、等级、四维、状态、技能、锁定、队伍顺序或成长随机数。

## 建议人工验收

这项静态 UI 建议玩家有空时用 GM 号做一次短验收：

1. 在 GM/QA 获取两只或更多 Lv1 蓝人龙，打开“宠物 → 成长”，确认左图立即出现四项 Lv1 物种内百分位，右图在 Lv1 没有伪造成长结论。
2. 记录一只宠物左图四项，分别升到 Lv5、Lv10、Lv20；左图必须完全不变，右图、成长/级、评级和预测140应随真实升级证据更新。
3. 选择一只 Lv1 4V 很好但实测成长较差的宠物，确认两张图如实分歧且没有合并总品质。
4. 展开“设置参考线”，把某项调到实际分位之上/之下；只允许人工建议改变，宠物等级、四维、状态和所在栏位不得改变。
5. 联网保存、重新登录后确认参考线保留；再打开“内挂设置 → 捕捉”，确认不存在 Lv20 成长门槛、待处理或自动练宠暗示。

通过标准：玩家能独立回答“它出生四维是不是同物种极品”和“它练级成长是否值得继续”，且任何查看、调整或保存都不会处置宠物。

## 非目标

- 不开放自动训练、自动移动或自动删除已培养宠物。
- 不把 Lv1 4V 与隐藏成长关联，不公布隐藏底档或真实未来逐级结果。
- 不重掷旧宠、不迁移宠物实例、不操作真实玩家数据或共享 MySQL。
- 不在本片设计转生数值；下一项按路线图进入 P1.2 普通宠 0→1→2 转与重置平衡。
