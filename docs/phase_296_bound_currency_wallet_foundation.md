# Phase 296：绑定/非绑定四钱包兼容地基

## 结论

P1.2c 的第一小块先建立付费洗宠需要的货币事实，不提前实现价格、扣款或重置按钮：

- 石币和钻石各自拥有绑定、非绑定两个独立余额；
- 历史 `stoneCoins`、`diamonds` 明确定义为非绑定余额，继续允许市场、交易和市场货款邮件使用；
- 新增 `boundStoneCoins`、`boundDiamonds`，旧档缺失时按 `0` 读取；
- 绑定余额不能存入现有石币银行、购买玩家挂单、作为挂单计价货币或进入面对面交易；
- 不批量改写旧档，不改变旧余额价值，不增加 profile revision；
- 当前系统奖励来源暂不重新分类，仍沿用原字段。哪些来源以后发绑定币属于 P1.6 长线经济规则，不能借本次兼容改动静默改变。

这解决了后续“不同宠物使用不同货币/价格洗宠”的数据前提，同时没有把充值、真实人民币或正式支付 provider 混进宠物重置事务。

## 用户已经确定的付费重置边界

本阶段用项目 Pet Design Contract 固化了后续 P1.2c 的产品决定：

- 捕捉宠、普通宠、商业宠、奖励宠、进化宠和融合宠均可反复完整重置；不能只给商业宠开放；
- 只允许 `1转/2转 → Lv1、0转`，不提供 `2转 → 1转`；
- 成功后解除宠物绑定；价格和支付货币按宠物形态由 GM 配置；
- 成功重置永不返还费用、MM、喂石和练级时间；
- 所谓“失败回滚”只指技术事务失败：扣款、宠物重置、解绑、永久重置次数和审计必须全部提交，否则全部不发生；
- 不重抽实例 ID、形态、不可变 Lv1 4V、基础隐藏成长或已有主动/被动技能。

验证合同位于忽略目录 `.run/pet-design/p1_2c_paid_rebirth_reset_wallet_foundation.json`，不会作为运行时配置提交。

## StoneAge 8.0 基线与 Beastbound 差异

本地稳定 StoneAge 8.0 参考中没有可直接照搬的“绑定/非绑定四钱包 + 所有宠物付费洗转生”标准机制；`npc_transmigration.c` 负责服务端转生选择、条件和确认，宠物仓库中的“洗宠”字样则是异常/重复宠物清理，不是玩家反复付费重置。

因此这是 Beastbound 面向私服式长期运营的原创差异。为了避免过度设计，玩家仍只看到石币和钻石两类货币；“绑定”只是同一种货币的交易属性，不新增第三、第四种货币名称，也不新增独立系统页。

## 数据合同

| 货币 | 非绑定字段 | 绑定字段 | 可进入玩家交易 | 旧档处理 |
| --- | --- | --- | --- | --- |
| 石币 | `stoneCoins` | `boundStoneCoins` | 仅非绑定 | 旧 `stoneCoins` 原值保留，绑定补 0 |
| 钻石 | `diamonds` | `boundDiamonds` | 仅非绑定 | 旧 `diamonds` 原值保留，绑定补 0 |

Node 新增 focused `currency-wallet.js`，统一货币/绑定轴、字段映射、余额读取、限额、写入、可交易余额和总余额。现有市场、银行、邮件货款与面对面交易继续通过非绑定读取器，避免以后某个调用者误把“总余额”当成可交易余额。

Godot 新增对应的 `currency_wallet_model.gd`；`PlayerProgressModel.currency_amount()` 继续明确返回可交易余额，另提供绑定余额和总余额读取。旧本地 schema1 档会补两个零字段；服务端 schema3 档仍保持不丢未来字段的 opaque 规则。

## 旧存档与 MySQL

没有 profile schema 升级或批量迁移：

- 新账号持久化四个字段；
- 旧账号的公开响应会补两个绑定零余额，但内部档案不因此写回；
- 第一次真正发生绑定货币变动时，专用权威事务才写入对应字段；
- MySQL 继续把 profile 作为现有 JSON 文档增量保存，不新增表、不重写全服账号；
- profile 迁移资产摘要已把两个绑定余额纳入校验，未来迁移不能漏币或把绑定币改成非绑定币。

## 玩家交易边界

- 现有银行只有非绑定石币；绑定石币不能借银行转换属性。
- 玩家市场只接受 `stoneCoins` 或 `diamonds`。提交 `boundStoneCoins` / `boundDiamonds` 会明确返回“绑定货币不能用于玩家交易”，不再默默降级为石币。
- 买家即使绑定余额充足，只要对应非绑定余额不足，购买仍失败且挂单、背包、四个余额都不变。
- 面对面交易的石币仍只读取 `stoneCoins`；绑定石币不能补足报价。
- 市场成交后的邮件货款仍是原买家支付的非绑定货币，领取只增加卖家的非绑定余额，绑定余额保持不变。
- 玩家邮件当前不支持直接附带货币，因此没有额外的绑定币发送入口。

## 非目标与下一步

本阶段没有：

- 决定任务、战斗、挂机、活动或充值分别发绑定还是非绑定货币；
- 编写每种宠物的重置价格；
- 实现扣款、宠物解绑、重置计数、永久审计、HTTP 接口或确认 UI；
- 接入人民币支付、退款或充值 provider。

下一小项 P1.2c-2 将建立 GM 可配置的全形态重置价格目录：所有宠物都有明确规则，普通捕捉宠默认用石币，稀有/奖励/进化/融合/商业档可使用钻石或更高价格；配置还会明确能否使用绑定余额及扣款顺序，代码中不藏人民币价格。
