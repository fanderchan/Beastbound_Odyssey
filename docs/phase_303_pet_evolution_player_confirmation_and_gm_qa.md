# Phase 303：宠物进化玩家确认、GM 验收与实机证据

## 结论

P1.3d 已完成进化流程的玩家确认层和 GM/QA 验收层，但没有提前开放生产进化：

- 玩家成长页只显示服务端确认的源宠、P90 成长战力门槛、材料、绑定优先石币拆款，以及“哪些会重抽、哪些会保留”；
- 第一次点击只进入明确的二次确认态，第二次才提交进化；结果未知时保留同一个 operation ID，明确失败才允许重新生成；
- GM 可一次准备乌力、风狐各一只未达 P90 与一只已达 P90 的真实 authority-v1 一转 Lv140 样本，并补齐两条路线所需资格、材料和绑定石币下限；
- GM 样本使用正式新宠 factory、升级结算、转生周期和资格判断，不用手写属性假装达标；清单存在后不会重抽、覆盖或补发已删除样本；
- 两条路线仍为 `runtimeEnabled=false`、正式资源状态 `deferred`。玩家不会看到入口，GM 准备样本也不会绕过正式美术门禁或消耗真实玩家资产。

## StoneAge 8.0 基线与 Beastbound 取舍

石器时代 8.0 的成熟基线是高价值宠物转换由服务端检查前置、材料和宠物状态，并在玩家确认后一次结算。Beastbound 沿用这条边界，没有增加玩家必须理解的技术恢复页面。

本项目的差异是进化后二代 Lv1 4V 与隐藏成长都会重新抽取，同时保留源宠 0转/1转公开成长履历。因此二次确认不能只写“确定进化吗”：它必须明确提醒结果可能变强也可能变弱，并分别列出重抽项、保留项和终局互斥。这个额外说明直接保护高价值宠物投入，不是为了堆功能；COMMIT、receipt、revision、审计 ID 和私有随机身份仍不进入玩家界面。

## 玩家确认合同

成长页仅在以下本地快筛全部满足时请求报价：服务端账号、成长页、指定源 form、一转 Lv140、authority-v1、未进化、路线运行开关已开且正式资源门禁为 `formal`。客户端快筛只是避免无意义请求，最终资格、P90、材料、钱包和宠物状态仍由服务器重新计算。

只读报价采用严格白名单：

- `pet`：实例、源形态、Lv140/一转、实际成长战力、同形态 P90 固定门槛；
- `result`：目标形态、回到 Lv1/一转、重抽 Lv1 4V、重抽隐藏成长、保留 0/1 转履历和终局页签；
- `cost`：每种材料的需求/持有量、石币总额、绑定/非绑定余额与绑定优先实际拆款；
- 不接受额外字段，不接受私有 seed/roll，不接受与本地目录、当前宠物或服务端 profile revision 不一致的报价。

玩家看到的核心文案为：

1. `高防乌力｜1转 Lv140 → 晶甲乌力｜1转 Lv1`；
2. `成长战力 1410 / 门槛 1345（同形态前10%）`；
3. 材料和 `300,000` 石币的绑定/非绑定实际扣款；
4. `重新抽取：二代 Lv1 血、攻、防、敏与天生成长；结果可能变强，也可能变弱`；
5. 保留同一实例、名字、绑定/锁定、强化、技能、一转加成及 0转/1转成长履历；
6. 回到 Lv1/一转并进入“2转/进化/融合”终局，不能普通二转或作为融合材料；
7. 条件齐全时 100% 成功，失败不扣宠物、材料或石币。

第一次点击只把按钮改为“再次确认：消耗材料并重新抽取二代”，不触发网络提交。第二次点击才发送 `POST /pets/evolution`。等待期间按钮锁定；revision/目录冲突会清除旧报价并重新读取，绝不会自动提交。COMMIT 结果不明会保留原 operation ID，避免重试时二扣材料或重抽二代。

成功后客户端只应用服务端返回的权威档案，并同时失效旧进化报价和付费重置报价。切换宠物、账号或离开确认态都会废弃旧显示，但不会错误丢弃一个结果未知操作的幂等身份。

## GM/QA 验收档

新增独立命令 `gm_pet_evolution_qa` 和 `POST /gm/pets/evolution/qa`。共享本地 QA 策略升级为 `local_qa_full_v3`，仍只允许 `auth1373`，精确目录为 server `12`、client `31`、server-authoritative client `9` 项；没有通配符，也没有扩大普通玩家权限。

首次执行会在一个持久事务候选中：

- 为高防乌力和高地风狐分别生成未达 P90/已达 P90 两只一转 Lv140 样本，共四只；
- 每只都通过正式 Lv1 factory、139 次升级、一次正式转生重启、再 139 次升级，并用实际 `inspectPetEvolutionEligibility` 复核；
- 只在有四个宠物空间时创建，队伍放不下才进入兽栏；不覆盖任何已有实例；
- 发放两条一次性资格、两次路线合计材料（共鸣兽核 16、岩晶甲片 12、月岚尾羽 12）并把绑定石币补到 600,000 下限；已有数量更高时不减少；
- 写入稳定 manifest。再次执行只补足资格/材料/石币下限并刷新当前判断，不重抽、不覆盖，也不补发被玩家删除的样本。

GM 状态显示四只样本的实际成长战力/门槛、预期与实际判断、三种材料、绑定石币以及两个正式资源门禁。响应和界面都不显示 private seed/roll、operation ID 或底层存储信息。生产门禁关闭时会明确写“未开放，不会消耗玩家资产”。

## 可选人工验收

当前自动证据足以验收 P1.3d 的合同和界面，不要求用户现在手工测试。正式美术完成、路线门禁改为 `formal` 并打开生产开关后，按以下步骤做一次真实账号验收：

1. 用本地 QA 工具续期/初始化 `auth1373` 的 `local_qa_full_v3` 授权，启动本地后端后登录；
2. 打开 GM/QA，点击“进化验收档”，确认乌力/风狐各出现一只未达 P90 和一只已达 P90，实际判断 `4/4` 符合预期，且生产门禁状态准确；
3. 选择未达标乌力进入宠物成长页：不应出现可提交入口，服务器主动报价应返回成长战力未达同形态前 10%；
4. 选择达标乌力：核对源/目标、一转 Lv140、实际 P90 数字、三类消耗、绑定优先石币拆款以及重抽/保留/终局文案；
5. 第一次点击后确认宠物、材料、石币均未变化，按钮只进入二次确认；
6. 第二次点击会真实消耗 QA 账号资产。成功后确认仍为同一实例，变为晶甲乌力 Lv1/一转，Lv1 4V与隐藏成长均为新抽，一转页仍可回看源高防乌力 Lv140 实绩；
7. 在提交等待期间模拟断线，再用同一操作恢复，确认只扣一次材料/石币且不会重抽第二只结果；
8. 回到 GM/QA 再点入口，确认不会补发或重抽四只样本，只补足低于验收下限的材料与石币。

通过标准：门槛两拒两放；第一次点击零资产变化；成功后实例/公开历史保留而二代品质独立重抽；断线重试不二扣；普通玩家永远看不到 GM、私有随机数或技术恢复字段。
