# Phase 311：骑乘方向、人物比例与粘合层返工

## 结论

Phase 310 的第一版骑乘画面被项目所有者明确否决，不能继续作为视觉基线。复现截图中同时存在三个问题：正面人物和芽耳布伊朝向相反、人物相对坐骑过小、人物整张贴在宠物背上且只靠矩形宠物区域盖腿，缺少可信的座位与材质连接。

本阶段已重制正背骑手关键姿势及 `ride_idle/ride_walk` 全部 28 张运行帧，人物与坐骑方向统一；配置比例从 `0.76 / 1.08 = 0.704` 提高到 `0.94 / 1.08 = 0.870`，并新增每只坐骑一次的鞍垫后层。现行绘制顺序为：

1. 贴地阴影；
2. 原宠物动作帧；
3. 芽耳布伊鞍垫后层；
4. 可复用四足跨坐骑手动作帧；
5. 原宠物近侧头身遮挡。

它仍没有生成任何“人物×芽耳布伊”整套烘焙组合图。人物动作按人物外观与骑乘体型复用，鞍垫/挂点/遮挡按坐骑配置；同类四足新宠不需要重画本人物的 28 张骑乘帧。

当前自审通过，但用户视觉验收仍为 `pending`，P2.2b 不能据此勾选完成。

稳定本地参考 `/Users/fander/projects/_local_references/StoneAge/gmsv/data/ride.txt`、`leaderride.txt` 和相关 `ridePetTable/rideNo` 代码表明，成熟石器客户端为人物与骑宠组合选择专用骑乘编号，因此方向、人物比例和遮挡天然在一张组合动作里解决。公开骑乘资料也能看到人物、鞍位和宠物始终共享同一运动轴：

- https://pooyas.com/index.php?document_srl=9706&mid=sa_guide

Beastbound 不照搬组合图数量爆炸，但不能把“运行时分层”当作降低成片标准的理由。保留通用分层的前提是最终画面必须恢复石器成熟方案已经解决的四件事：同向、合适比例、明确座位、可信前后穿插。

## 返工内容

### 骑手动作

- 正面母版改为头、胸、骨盆、双手统一朝画面左下，和芽耳布伊正面母版同向；水平镜像后共同朝右下。
- 背面母版统一朝画面右上；水平镜像后共同朝左上。
- 髋部低坐，双腿包住不可见的宽背四足兽，双手握靠近鬃毛位置的短把手；不再沿用站立人物缩小后悬浮在背上的读法。
- `ride_idle` 正背各 6 帧；`ride_walk` 正背各 8 帧，共 28 张 256×256 运行帧。
- 背面行走首稿因远侧腿在部分帧消失被退回。最终稿要求源动作帧始终保留两条完整连接的腿与靴，遮挡只在运行时发生，不能在源素材中用“肢体消失”偷解穿插。
- 四套原始色键图、去色键图、处理参数与提示词均保存在人物资产目录；所有处理结果 `edge_touch_frames=[]`。

### 坐骑接口层

- 芽耳布伊新增正背各一张 256×256 鞍垫后层：深青叶纤维软垫、橄榄编边和克制皮革包边。
- 鞍垫绘制在宠物与人物之间，提供明确坐面和材质过渡；它是“每只坐骑一次”的资产，不随人物外观重复。
- 资源合同、来源权属、原始图、处理参数和替换路径登记在 `assets/mounts/bui_novice_sprout_earth5_wind5/`。
- 复杂坐骑以后可以在同一合同下补前侧皮带或专用近景遮挡层，但不能退回人物×宠物全组合烘焙。

### 自动质量门槛

`character_mount_art_check.gd` 现在额外检查：

- 两个视角都能加载鞍垫后层；
- 每张骑手帧与 256×256 边缘至少保留 4px 安全距离，避免动作被格子切断；
- 骑手按实际透明包围盒和最终缩放计算后的高度，不得低于坐骑的 `0.82`；
- 鞍垫接口在资产 manifest 中唯一登记，来源为项目原创 AI 辅助；
- 人物用户验收状态继续保持 `pending`，没有因自动测试通过而冒充用户认可。

## 验收步骤

1. 先看正面与背面截图。通过标准：人物和芽耳布伊朝向一致；人物不再像小人偶；髋部落在鞍位，腿从宠物两侧下垂；鞍垫只负责连接，不抢宠物造型。
2. 再看 14.4 秒录像。通过标准：八个逻辑方向切换时人物与宠物同时镜像；待机/行走不闪腿、不切靴、不漂鞍；上下起伏像骑手吸收四足步态而不是人物独自蹦跳。
3. 如果比例、鞍垫颜色或座位仍需调整，只改本坐骑档案和接口层；如果人物身份或跨坐姿势不认可，退回四足骑手母版，不扩产其他坐骑。

## 保留限制

- 当前仍只有正背两套独立母版，经镜像形成四种视觉朝向；八个逻辑方向不等于八套独立视觉视角。
- 芽耳布伊只是四足跨坐样板；双足肩骑、飞行背带和蛇形平台仍未生产。
- 这次只修正骑乘美术，不改变骑乘资格、移动、战斗属性、服务端数据或玩家存档。
