# Phase 319：宠物美术与动画历史经验技能化

日期：2026-07-19

## 目标

项目所有者要求把芽耳布伊首轮生产、骑乘方案返工和战斗审片得到的经验固化到技能中，使下一只宠物从自然语言设计开始就能得到完整、可验证且不会重犯旧问题的生产计划。

本阶段只更新开发技能、设计合同和路线说明，不生成新宠物，不修改运行时代码、战斗公式、玩家档案或数据库。当前芽耳布伊回避/骑乘战斗动画仍按 Phase 318 的结论保持视觉未通过，不能因技能更新而改记为完成。

## 固化的核心判断

### 世界与战斗方向分开

- 世界移动需要人物、宠物、每个正式支持的人物骑宠组合三类视觉主体。
- 三类主体均使用南、西南、西、西北、北、东北、东、东南八个独立源方向；最低 `idle 1 + walk 4`，每类 40 帧。
- 不再把两套母版、逻辑八方向或水平镜像称为真八向。
- 当前固定 10V10 战斗只绘制敌我两个三分之四斜向，所以战斗动作生产正、背两个正式视角；不把每个战斗动作无意义扩成八方向。

### 骑乘是独立整图主体

- Beastbound 当前合同为 AI 直接生成完整“人物骑着宠物”画面。
- 人物、宠物、座位、手脚、骑具和前后遮挡必须在同一张图中成立；运行时和离线均禁止重新拼合人物 PNG 与宠物 PNG。
- 禁止镜像补方向、删掉被遮挡肢体、鞍垫补缝和矩形前景遮罩。
- 未生产的组合显示徒步人物，不猜成其他坐骑，也不恢复旧分层方案。

### 战斗按语义验收

正式生产合同覆盖普通攻击、技能、防御受击、普通受击/恢复、回避、回避后反击、反击、负伤归位、击飞、可复活昏厥、复活和合击。

不是每种语义都必须机械地新增一套身体帧：盾面、冲击光、击飞轨迹和眩晕光环可以是独立运行时特效。但关闭消息/日志后仍必须一眼读懂行为；做不到就不能用复用动作掩盖。历史上已经明确拒绝：

- 用健康 `walk` 代替回避或负伤归位；
- 用整体骑乘 `idle/walk` 代替攻击、防御、回避和反击；
- 用普通 `hurt` 代替防御承压；
- 用闭眼微笑的睡姿代替可复活昏厥；
- 反击、技能和合击仅靠文字区分。

### 先小门槛，再扩产

先锁身份与比例，再验独立关键姿势，随后才生产真八向和完整战斗矩阵。第一只宠物的联系表或实机录像未通过时，不扩产第二只宠物。自动检查只证明结构和接线；风格、比例、重量、冲击感和动作节奏必须保留项目所有者截图/录像验收。

## 技能更新

### 项目宠物设计技能

`.agents/skills/design-beastbound-pets/` 现支持两种明确范围：

- 概念/数值设计：`artStatus=deferred`，只记录未来美术简报；
- 正式美术生产：必须提供 `presentation.artProduction`，并转入专门美术导演流程。

新增的美术合同会校验三类世界主体、真八方向、世界动作、两个正式战斗视角、完整战斗语义、至少一个支持人物、AI 整图骑乘、禁镜像/禁分层、必录审片场景和项目所有者验收要求。

详细合同：`.agents/skills/design-beastbound-pets/references/art-animation-production.md`。

### 美术导演技能

全局 `$stoneage9-art-director` 已加入 Beastbound 路由：先识别目标项目，再读取本项目的美术合同。通用分层骑乘建议不再覆盖 Beastbound 已确认的 AI 整图路线。宠物动作标准和可选执行简报同步加入真八向、整图骑乘、完整战斗语义与真实 Godot 审片门槛。

## 后续使用

下一次自然语言宠物请求仍先完成玩法、成长、投放、技能和经济合同。若请求包含“正式、完整、可发行、可骑、素材或动画”，流程会继续完成美术生产矩阵；若只是讨论数值和定位，则不会擅自启动大型图片生产。

第一个新宠物仍应分阶段交付：身份/关键姿势截图 → 真八向世界录像 → 双视角战斗动作联系表 → 隔离 10V10 动作导演录像 → 项目所有者验收。这样既不会遗漏完整性，也不会在审美方向错误时浪费整批生成成本。
