# Phase41：治疗 / 休息恢复

## 目标

这一阶段让宠物生命值从“战斗后只写回损耗”推进到可恢复。

## 规则

- 宠物面板新增短按钮 `治疗`。
- `治疗` 会把当前选中的宠物生命恢复到 `maxHp`。
- `治疗` 不改变宠物状态、经验、技能、属性、队伍位置或兽栏位置。
- 兽栏里的宠物允许被 `治疗`。
- 队伍里状态为 `休息` 的宠物，会在世界界面定时恢复生命。
- 兽栏宠不参与休息恢复。兽栏本身不是休息状态，取出后统一回到 `待机`。
- 休息恢复不会自动把宠物从 `休息` 改为 `待机` 或 `战斗`。
- 休息恢复不会自动指定主宠。

## 恢复数值

- 世界界面每 5 秒执行一次休息恢复。
- 每次恢复 `max(1, 最大生命 5%)`。
- 恢复不会超过 `maxHp`。
- 恢复 tick 不主动刷消息，避免干扰玩家操作。

## UI

- 宠物底部按钮保持短文案：`战斗` / `待机` / `休息`、`治疗`、`存入` / `取出`、`改名`。
- 宠物详情继续在滚动区显示生命变化。
- 按钮区位置不跟随详情文字变化。

## 自动测试命令

```sh
godot --headless --path client/godot --scene res://scenes/Main.tscn --quit-after 1800 -- --auto-pet-recovery-check
```

覆盖：

- 兽栏宠可以通过 `治疗` 回满生命。
- 治疗不改变兽栏状态。
- 休息宠会通过恢复 tick 增加生命。
- 兽栏宠不会通过休息恢复 tick 增加生命。
- 待机宠不会通过休息恢复 tick 增加生命。
- 休息恢复不会刷世界消息。
- 打开宠物面板时，恢复后的生命会刷新到详情区。
