# Phase44：真正的宠物图鉴列表

这一阶段把 `图鉴` 从宠物详情里的切换页，升级为世界 HUD 的独立入口。

## 规则

- 操作栏新增 `图鉴` 按钮，按钮文字保持短，不挤压宠物状态按钮文案。
- 图鉴列表展示模板库里的全部宠物形态。
- 已捕捉或当前持有的形态显示真实名称和完整资料。
- 已遇见但未捕捉的形态显示真实名称和完整图鉴资料，但记录为 `已遇见`。
- 未遇见的形态在列表和详情里都显示为 `？？？`，不泄露内部 `formId`。
- 战斗结算会把敌方宠物形态记录为已遇见。
- 捕捉成功会把对应形态记录为已捕捉，并显示当前持有数量。

## 手动自测

打开游戏，并直接进入 Phase44 图鉴列表预览：

```bash
godot --path client/godot --scene res://scenes/Main.tscn -- --pet-codex-list-preview
```

打开游戏，从正常入口自己点：

```bash
godot --path client/godot --scene res://scenes/Main.tscn
```

进入后点右下操作栏的 `图鉴`，可以测试列表选择、关闭、再打开，以及和 `宠物` 面板互斥。

## 自动回归

```bash
godot --headless --path client/godot --scene res://scenes/Main.tscn --quit-after 1800 -- --auto-pet-codex-list-check
```

这个检查覆盖：

- 世界 HUD 上存在短按钮 `图鉴`。
- 图鉴列表数量等于宠物模板形态数量。
- 默认已持有的布伊形态显示为已捕捉，并统计持有数量。
- 未遇见形态被遮罩为 `？？？`。
- 手动记录遇见后会显示 `已遇见` 和图鉴资料。
- 战斗逃跑会记录遇见。
- 战斗捕捉会记录已捕捉。
- 图鉴详情不显示内部 ID。
